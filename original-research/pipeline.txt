### Voice-to-Text Pipeline with Speaker Diarization

- **Core Models**: Use OpenAI's Whisper large-v3 for accurate English transcription and NVIDIA's NeMo MSDD for robust speaker diarization, handling multiple speakers and overlaps effectively.
- **Audio Handling**: Convert all input to mono format and chunk large files into manageable segments to support extended conversations without performance issues.
- **Speaker Management**: For unknown speakers, generate placeholder labels (e.g., Speaker_1) and extract audio samples for potential future training; include an optional flag to bypass fingerprinting if not needed.
- **Local Execution**: Fully offline on Mac M2 using free Python libraries, leveraging Apple Silicon for efficient processing.

#### Overview
This pipeline processes audio files into transcribed text with speaker labels, optimized for English conversations with multiple participants. It ensures high-quality results even with overlapping speech by integrating transcription and diarization models. All components run locally on a Mac M2, requiring no internet or paid services.

#### Dependencies
- Python 3.10+.
- OpenAI Whisper (for transcription): Install via `pip install openai-whisper`.
- NVIDIA NeMo (for diarization): Install via `pip install nemo_toolkit[asr]`.
- Additional libraries: pydub (for audio manipulation), librosa (for chunking and processing), FFmpeg (for format conversions, install via Homebrew).

#### Input and Output
- **Input**: Audio file (e.g., WAV, MP3) of any length, potentially stereo.
- **Output**: JSON or text file with timestamped transcriptions, speaker labels, and optional extracted speaker samples (short audio clips for unknowns).

#### Pipeline Steps
1. **Audio Preprocessing**: Convert stereo to mono by averaging channels. Chunk large files into 30-second segments with 5-10 second overlaps to handle long conversations.
2. **Transcription**: Use Whisper large-v3 to transcribe each chunk in English-only mode, generating text with timestamps.
3. **Diarization**: Apply NeMo MSDD to identify speakers, extracting embeddings and clustering for labels, supporting overlaps.
4. **Speaker Handling**: Assign placeholders for unknowns; extract 5-10 second samples per unique speaker. Align with transcriptions.
5. **Post-Processing**: Merge chunk results, resolve overlaps, and output formatted transcript.

#### Configuration Flags
- `--no-fingerprint`: Skip extracting samples for unknown speakers, using only placeholders.

#### System Requirements and Setup
Ensure a Mac M2 with at least 16GB RAM for optimal performance, as larger models like Whisper large-v3 require ~10GB VRAM. Install Python 3.10 or higher. Use Homebrew to install FFmpeg (`brew install ffmpeg`) for audio handling. Dependencies include:
- **OpenAI Whisper**: Provides the large-v3 model (1550M parameters) for multilingual speech recognition, but configured here for English-only to optimize accuracy. Install with `pip install openai-whisper`.
- **NVIDIA NeMo**: Supplies the MSDD (Multi-Scale Diarization Decoder) for neural-based speaker diarization, handling voice activity detection, embedding extraction, and overlap-aware labeling. Install via `pip install nemo_toolkit[asr]`; enable MPS fallback for Mac compatibility with `os.environ["PYTORCH_ENABLE_MPS_FALLBACK"] = "1"`.
- **Supporting Libraries**: pydub for mono conversion and basic manipulation (`pip install pydub`); librosa for chunking and audio analysis (`pip install librosa`).

This setup ensures all processing occurs locally, leveraging Apple Silicon's Neural Engine for efficiency.

#### Input Specifications
- Audio formats: WAV, MP3, or others convertible via FFmpeg.
- Assumptions: English language; multiple speakers (no fixed limit, as MSDD dynamically clusters); potential overlapping speech.
- Handling large files: Automatically detect file size; if exceeding 30 seconds, apply chunking to prevent memory overflow.

#### Detailed Pipeline Steps
The pipeline is modular, allowing sequential execution in Python. Each step builds on the previous, with outputs passed as intermediates (e.g., audio chunks, timestamps).

1. **Audio Preprocessing**:
   - Load the input audio file.
   - Convert to mono: If stereo, average channels using pydub's `set_channels(1)` or torch/torchaudio's mean across dimensions.
   - Normalize sample rate to 16kHz (Whisper's default) and bit depth to 16-bit PCM.
   - For large conversations: Split into 30-second chunks with 5-10 second overlaps to maintain context across boundaries. Use librosa to slice arrays, ensuring no mid-word cuts by aligning on silence where possible.

2. **Transcription with Whisper large-v3**:
   - Load the model: `model = whisper.load_model("large-v3")`.
   - Transcribe each mono chunk: `result = model.transcribe(chunk, language="en")`, focusing on English for better performance (avoids multilingual overhead).
   - Output: Dictionary with text segments and timestamps.
   - Handle chunks: Concatenate results, adjusting timestamps for overlaps by merging duplicate phrases.

3. **Speaker Diarization with NeMo MSDD**:
   - Initialize MSDD via NeMo's `NeuralDiarizer` with a configuration for multi-speaker scenarios (e.g., `diar_msdd_telephonic` pretrained model).
   - Process chunks: Extract voice activity (VAD with MarbleNet), speaker embeddings (TitaNet), and cluster dynamically.
   - Overlap handling: MSDD uses multi-scale embeddings and neural decoding to estimate probabilities for overlapping speakers, supporting best-effort fingerprinting.
   - Output: RTTM files with speaker timestamps and labels.

4. **Integration of Transcription and Diarization**:
   - Align Whisper timestamps with NeMo speaker labels: Map segments using overlap scoring to assign speakers to words.
   - For unknown speakers: Use placeholders (e.g., Speaker_1, Speaker_2). Extract samples by segmenting audio based on diarization timestamps (5-10 second clips per unique cluster) for future training. Store in a directory with labels.
   - If `--no-fingerprint` flag is set, skip sample extraction but retain placeholders.

5. **Post-Processing and Output**:
   - Merge all chunk results into a single transcript.
   - Format output: Timestamped text with speaker prefixes (e.g., "[00:01] Speaker_1: Hello...").
   - Export as JSON (for structured data) or plain text; include speaker samples folder if applicable.

#### Configuration and Flags
- **English-Only Mode**: Enforce via Whisper's `language="en"` parameter.
- **No-Fingerprint Flag**: Add a command-line argument to toggle sample extraction, defaulting to enabled for comprehensive output.
- **Customization**: Allow parameters for chunk size, overlap duration, and max speakers (though MSDD handles dynamically).

#### Performance Considerations on Mac M2
- Whisper large-v3 runs at ~1x real-time on M2; use `turbo` variant for faster processing if accuracy trade-off is acceptable.
- NeMo MSDD may fallback to CPU for unsupported operations; expect 2-3x real-time for diarization on moderate files.
- Test with diverse audio to tune for accents or noise; MSDD's overlap-aware design mitigates over-talking issues.
