# Research Findings - Day 1 (2026-02-18)

## Overview
Initial research on voice transcription with speaker identification/voice fingerprinting. Focus on open-source solutions that can run locally on Mac M2 32GB.

---

## Key Discoveries

### 1. The Tech Stack Pipeline

Voice fingerprinting + transcription involves **three main components**:

1. **ASR (Automatic Speech Recognition)** - Convert speech to text
2. **Speaker Diarization** - Identify "who spoke when" 
3. **Voice Embedding/Fingerprinting** - Create unique voice signatures

### 2. Leading Open-Source Solutions

#### **WhisperX** (Top Pick for Integration)
- GitHub: https://github.com/m-bain/whisperX
- Combines OpenAI Whisper + speaker diarization
- **70x real-time transcription** with batched inference
- Word-level timestamps via wav2vec2 alignment
- Uses pyannote-audio for speaker identification
- Works on GPUs with <8GB memory
- Can run on CPU (slower but viable on Mac M2)

#### **Whisper Diarization** (MahmoudAshraf97)
- GitHub: https://github.com/MahmoudAshraf97/whisper-diarization
- 5.4k stars, 496 forks
- Combines Whisper + Voice Activity Detection (VAD) + Speaker Embedding
- Uses Demucs for vocal extraction
- Uses TitaNet for speaker embeddings
- Uses ctc-forced-aligner for timestamp correction
- **BSD-2-Clause license** (commercial-friendly)

#### **Pyannote.audio** (The Speaker ID Engine)
- The gold standard for speaker diarization
- Version 3.1 is latest (2024-2025)
- Pipeline: VAD → Segmentation → Speaker Embedding → Clustering
- Requires HuggingFace account and token
- Can distinguish speakers with high accuracy
- Handles overlapping speech (with limitations)

### 3. Speaker Diarization Process

```
Audio Input
    ↓
Voice Activity Detection (VAD) - Remove silence
    ↓
Segmentation - Split into speaker turns
    ↓
Speaker Embedding - Create voice fingerprints (x-vectors/i-vectors)
    ↓
Clustering - Group similar embeddings by speaker
    ↓
Output: "Speaker A: [transcript], Speaker B: [transcript]"
```

### 4. Voice Fingerprinting Technologies

| Technology | Description | Use Case |
|------------|-------------|----------|
| **i-vectors** | Intermediate vectors, statistical model | Traditional speaker recognition |
| **x-vectors** | Deep learning embeddings (TDNN-based) | Modern speaker recognition |
| **ECAPA-TDNN** | Emphasized Channel Attention, Propagation & Aggregation | State-of-the-art embeddings |
| **TitaNet** | NVIDIA's speaker embedding model | High accuracy speaker ID |
| **Resemblyzer** | Voice encoder for similarity comparison | Deepfake detection, voice cloning |

### 5. Commercial Services Analysis

#### **Otter.ai** (The Benchmark)
- Proprietary ASR pipeline
- Speaker identification algorithm learns from tagged paragraphs
- Cloud-based processing
- Claims high accuracy but struggles with:
  - Overlapping speech
  - Background noise
  - Speaker misattribution
- **Pricing:** ~$10-20/month

#### **AssemblyAI** (API-First)
- Universal-2 model with speaker diarization
- 10.1% improvement in Diarization Error Rate (DER) in 2024-2025
- 30% better performance in noisy environments
- Can handle speaker segments as short as 250ms
- API-based, not local

#### **Deepgram** (Speed-focused)
- Real-time streaming with low latency
- Nova-2 model with speaker diarization
- Good for real-time applications

### 6. Mac M2 32GB Compatibility

**GOOD NEWS:** Several options can work on Mac M2:

1. **Whisper (base models)** - Runs on M1/M2 CPU
2. **Whisper.cpp** - C++ port optimized for Apple Silicon
3. **Faster-Whisper** - Uses CTranslate2, efficient on CPU
4. **Pyannote** - Can run on CPU (slower but works)

**Performance Expectations on Mac M2:**
- Whisper base/small: Real-time or faster
- Whisper medium: ~0.5-1x real-time
- Whisper large: ~0.2-0.5x real-time (slower but higher quality)
- Speaker diarization: Adds 20-50% processing time

## Key Research Questions Answered

1. **Can this run on Mac M2 32GB?** 
   - YES: Whisper (base/medium), Whisper.cpp, faster-whisper, pyannote (slower)
   
2. **Free or one-time cost?**
   - Multiple open-source options (BSD-2, MIT, Apache licenses)
   - No API costs for local processing
   - Requires initial setup time

3. **How do commercial services do it?**
   - ASR + Speaker Embedding + Clustering pipeline
   - Cloud processing for speed
   - Proprietary training on millions of hours
   - Trade-off: convenience vs. privacy/cost
